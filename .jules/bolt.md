## 2024-05-24 - Eager Loading in Client-Side Routing
**Learning:** EcoLoop uses a client-side routing approach where all HTML content for hidden tabs is rendered on the initial page load by `index.php`. This architectural choice means media elements (like external QR codes or marketplace item images) inside these hidden sections are eagerly loaded by the browser, causing unnecessary network requests and blocking the main thread during the initial load, which constitutes a significant performance bottleneck.
**Action:** When working on applications with similar client-side hiding/showing of pre-rendered content, always add `loading="lazy"` to media elements (like `<img>` and `<iframe>`) within the hidden sections to defer their loading until they actually become visible to the user.
